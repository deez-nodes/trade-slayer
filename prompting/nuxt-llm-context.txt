# Nuxt 4 Reference — LLM Context

## Overview
Nuxt is a Vue.js framework for building full-stack web applications. It provides file-based routing, auto-imports, SSR, and an opinionated directory structure.

## Directory Structure
```
app/
├── app.vue              # Root component (entry point)
├── assets/css/          # Processed stylesheets
├── components/          # Auto-imported Vue components
├── composables/         # Auto-imported Vue composables
├── layouts/             # Layout wrappers (use <slot /> for page content)
├── pages/               # File-based routing (each .vue = a route)
├── plugins/             # Nuxt plugins
├── utils/               # Auto-imported utility functions
server/
├── api/                 # Server API routes (Nitro)
├── middleware/          # Server middleware
public/                  # Static assets served as-is at /
nuxt.config.ts           # Root configuration
```

## Auto-Imports
Nuxt auto-imports from these directories — no import statements needed:
- `app/components/` → components available in all templates
- `app/composables/` → composables available in all scripts
- `app/utils/` → utility functions available in all scripts
- Vue APIs: `ref`, `computed`, `watch`, `onMounted`, etc.
- Nuxt composables: `useRoute`, `useRouter`, `useFetch`, `useHead`, `$fetch`, etc.

## Vue SFC Convention
```vue
<template>
  <div :class="$style.container">
    <p>{{ message }}</p>
  </div>
</template>

<script setup lang="ts">
const message = ref('Hello')
</script>

<style module>
.container {
  padding: 1rem;
}
</style>
```

## CSS Modules
- Use `<style module>` (not `<style scoped>`)
- Access classes via `$style.className` in templates
- Classes are auto-scoped to the component

## Components
Components in `app/components/` are auto-imported by filename:
- `app/components/StrategyCard.vue` → `<StrategyCard />` in any template
- Props defined with `defineProps<{ prop: Type }>()`
- Events defined with `defineEmits<{ (e: 'name', value: Type): void }>()`

## Composables
Functions in `app/composables/` are auto-imported by export name:
```ts
// app/composables/useLLM.ts
export function useLLM() {
  // returned object available anywhere via useLLM()
  return { chat, listModels }
}
```

## Pages & Routing
- `app/pages/index.vue` → `/`
- `app/pages/about.vue` → `/about`
- `app/pages/posts/[id].vue` → `/posts/:id` (dynamic)
- Pages render inside `<NuxtPage />` (usually in a layout)

## Layouts
- `app/layouts/default.vue` is the default layout
- Layouts use `<slot />` to render page content
- Set per-page layout via `definePageMeta({ layout: 'custom' })`

## Data Fetching
- `$fetch` — simple promise-based fetch (from ofetch, auto-imported)
- `useFetch` — SSR-aware composable with caching/hydration
- `useAsyncData` — SSR-aware wrapper for any async operation

For client-only API calls to local services, `$fetch` is preferred over `useFetch`.

## Nuxt UI
This project uses `@nuxt/ui` which provides pre-built components:
- `UApp` — root wrapper
- `UCard` — card container with #header, default, #footer slots
- `UCheckbox` — checkbox with label, v-model support
- `UDashboardGroup`, `UDashboardSidebar`, `UDashboardPanel` — dashboard layout
- `UDashboardNavbar` — top navigation bar
- `UNavigationMenu` — sidebar navigation
- All Nuxt UI components are auto-imported

## TypeScript
- `<script setup lang="ts">` in all SFCs
- Types can be defined inline in components
- `defineProps<T>()` for typed props
- `nuxt.config.ts` uses `defineNuxtConfig()`

## Project-Specific Conventions
- Package manager: pnpm (never npm)
- CSS: `<style module>` blocks with `$style.className` references
- No inline class strings in templates
- Minimal comments — code should be self-explanatory
